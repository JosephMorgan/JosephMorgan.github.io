Selenium的使用

Selenium百科：Selenium包括Selenium1.0（即Selenium RC）和Selenium2.0（Selenium1.0+ WebDriver）。Selenium1.0是通过JS注入方式与浏览器进行交互的，而Selenium2.0中为了更好的模拟用户真实环境，它直接利用了浏览器的nativesupport来操作浏览器的。Selenium1.0和WebDriver原属于两个不同的测试工具， Selenium1.0与WebDriver合并后，Selenium1.0提供了WebDriver不具备的多浏览器支持，而WebDriver提供了Selenium1.0中不具备的键盘和鼠标等系统级的操作以及对对话框的支持。

Ant百科：Ant是一个常用的构建工具，我们自动化过程中工程的编译、执行、发邮件等操作都可以通过构建工具来执行；我们只需要在构建文件（一个xml文件）中进行下简单的配置便可通过Ant对我们的工程进行构建。

TestNG百科：TestNG是一个涵盖所有类型的测试：单元，功能，集成等的测试框架，选TestNG是因为相对于JUnit而言，TestNG具备了JUnit所有功能，而同时又拥有很多JUnit不具备的特点，主要包括：

  • TestNG拥有一个xml配置文件，可以很方便的对所有用例进行管理、配置。
  • TestNG支持依赖测试。
  • TestNG参数化测试功能更加强大，既可以在case中指定参数，又可以通过配置文件进行传参。

Jenkins百科：Jenkins是一个持续集成工具，通过它可以监控自动化case代码变更情况、定时执行自动化测试脚本、生成测试报告、邮件通知负责人。

Selenium IDE百科：Selenium的IDE(集成开发环境)是一个易于使用的Firefox插件,用于开发Selenium测试用例。它提供了一个图形用户界面，用于记录使用Firefox浏览器，用来学习和使用Selenium用户操作，但它只能用于Firefox浏览器，不支持其它浏览器。它所记录的脚本可以被转换成多种编程语言的脚本，如Java、Python、C#等。

搭建环境

1、安装JDK及配置JDK环境变量

2、Eclipse安装，直接下载可用的Eclipse包即可。

3、Eclipse中安装TestNG插件，在Eclipse中选择help->install new software –> Add -> 弹出框中Name填TestNG（可填其它），Location填写http://beust.com/eclipse （此为插件地址）点击ok。

点击ok后Eclipse会自动的检测TestNG的插件内容，勾选要加载的内容，点击next。

点击next后就开始安装该插件，完成后重启eclipse使生效。Eclipse启动后，选择file->new->other，弹框中出现TestNG字样，表示安装成功。

4、Selenium2.0下载，去selenium官网下载对应语言版本的Selenium WebDriver包即可。

新建工程

1、新建一个Java工程，并在工程根目录下新建一个lib文件夹用来存放工程所需要用的jar包。

2、把下载的selenium2.0压缩包解压，将解压目录中的selenium-java-2.53.0.jar、selenium-java-2.53.0-srcs.jar和libs目录下所有jar文件复制到工程目录下的lib文件夹。

3、将lib文件夹中jar包添加到该工程build path中：选择所有jar包->右键后选择build path->点击add to build path完成jar包添加。

4、下载谷歌浏览器驱动程序chromedriver.exe（若使用其它浏览器，下载对应浏览器的驱动程序即可），并将该exe程序放到C:\Windows\System32目录下。

5、工程根目录下新建TestNG配置文件testng.xml（工程名右键，TestNG>Convert to TestNG）。

编写用例

示例

1、工程下新建一个包，并新建一个testng类，方法：右键包名->New->Other->testng->Testng Class，然后选择Next输入类名即可。

2、写用例，建议用Page Object设计模式。

3、用例写完后，可以看看效果。方法：鼠标放在要执行的用例的文件上右键，选择run as后，点击TestNG Test即可。

用TestNG

1、如果写了很多用例，那么可以用TestNG的xml配置文件来管理用例，只需将用例在TestNG配置文件testng.xml中进行相关配置即可。配置文件说明：

配置文件中suite标记一个测试单元，可以包含多个测试用例每个suite中的测试用例通过test标签来进行定义。参数parallel是多线程标记，通过thread-count来指定线程数,verbose是测试用例日志级别，包括0-10,0到10记录日志详细程度依次递增。classes标签中name值为前面所写的用例，配置规则：包名+“.”+类名。

2、TestNG配置文件执行，方法：鼠标放在testng.xml文件上右键，选择run as后，点击Testng Suite即可。

3、查看运行结果，case执行完后，刷新当前工程，会发现在工程根目录下出现test-output目录，该目录是TestNG自动生成的目录，case运行结果可在该目录下查看。进入test-output通过浏览器打开emailable-report.html文件即可看到case运行详细结果。
